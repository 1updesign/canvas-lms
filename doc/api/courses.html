Courses API

API for accessing course information.

Actions

GET /api/v1/courses.json CoursesController#index

Returns the list of active courses for the current user.

API method argument:

  • enrollment_type

    [optional, "teacher"|"student"|"ta"|"observer"|"designer"] When set, only return courses where the user is enrolled as this type. For example, set to "teacher" to return only courses where the user is enrolled as a Teacher.

  • include[]
    "needs_grading_count"
    Optional information to include with each Course.

    When needs_grading_count is given, and the current user has grading rights, the total number of submissions needing grading for all assignments is returned.

Example Response:

[ { 'id': 1, 'name': 'first course', 'course_code': 'first', 'enrollments': [{'type': 'student'}] },
  { 'id': 2, 'name': 'second course', 'course_code': 'second', 'enrollments': [{'type': 'teacher'}] } ]

API response field:

  • id

    The unique identifier for the course.

  • name

    The name of the course.

  • course_code

    The course code.

  • enrollments

    A list of enrollments linking the current user to the course.

  • needs_grading_count

    Number of submissions needing grading for all the course assignments. Only returned if include[]=needs_grading_count

GET /api/v1/courses/:course_id/sections.json CoursesController#sections

Returns the list of sections for this course.

[ 

  {
    "id": 1,
    "name": "Section A",
    "students": [...]
  },
  {
    "id": 2,
    "name": "Section B",
    "students": [...]
  }

]

API method argument:

  • include[]
    "students"
    Associations to include with the group.

Example Response:

?include[]=students

API response field:

  • id

    The unique identifier for the course section.

  • name

    The name of the section.

GET /api/v1/courses/:course_id/students.json CoursesController#students

Returns the list of students enrolled in this course.

Example Response:

[ { 'id': 1, 'name': 'first student', 'sis_user_id': null },
  { 'id': 2, 'name': 'second student', 'sis_user_id': 'from-sis' } ]

API response field:

  • id

    The unique identifier for the student.

  • name

    The full student name.

  • sis_user_id

    The SIS id for the user’s primary pseudonym.
